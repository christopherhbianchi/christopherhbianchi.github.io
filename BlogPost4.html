switch VS. if/else

Switch and if/else perform similar functions however the switch is usually a much more concise and efficient way to run multiple if/else statements. However, each has unique advantages and disadvantages, and is optimal in different situations.

if/else

We use if/else statements if we want our code to perform an action if the condition is true and if the condition is false.

Example using if/else:
if (answer == correctAnswer) {
score += 10;
System.out.println(“That’s right. You get 10 points.”);
} else {
score -=5;
System.out.println(“Sorry, that’s wrong. You lose 5 points.”);
}

When to use if/else:

    Simple comparisons
    If there aren't a lot of compares for a condition in your program

switch

In a switch statement the first line specifies the variable that is being tested. We then form our block of code using the curly brackets { }, with multiple cases. Then each case line checks the test variable in the switch statement against a specific value. Then when one of these values being checked in the case statement matches the variable being tested, the code then continues to run until it reaches a “break” statement, or it get’s to a default statement and then reaches the “break” statement. The switch statement works faster than if/else.

Example using switch:
switch (grade) {
case = ‘A’;
System.out.println(“You got an A! Awesome!”);
break;
case = ‘B’;
System.out.println (“You got a B! Beautiful!”);
Default:
System.out.println (“You got an F. You’ll do well in Congress!”);
}

When to use switch:

    Complex comparisons
    If there's a lot of compares for a condition in your program

In Conclusion

Switch and if/else perform similar functions however the switch is usually a much more concise and efficient way to run multiple if/else statements. However, each has unique advantages and disadvantages, and is optimal in different situations. It is up to the programmer to decide which is more appropriate for the program’s needs.
